## Top 8 Specific Weaknesses

### W1 — Single seed for all LLM training results (CRITICAL, UNCHANGED)
**Location:** Section 5 / Tables 3, 9  
All GRPO results — both MBPP 200×4 (+2pp) and 4-cycle HumanEval DAPO — are n=1 seed.
Binomial SE ≈ ±5pp at 100 tasks (MBPP), ≈ ±5.4pp at 82 tasks (HumanEval). Every
quantitative claim in these sections is statistically indistinguishable from noise without
multi-seed replication.  
**Fix:** EXP-038 (multi-seed MBPP GRPO, n=3) + EXP-080 (multi-seed 4-cycle HumanEval DAPO, n=3) — both blocked on A800.

### W2 — Tau2 router overfitting unexplained (NEW FINDING, CRITICAL, UNCHANGED)
**Location:** Section 7 (tau2 held-out) / Table 7  
Routing accuracy drops from 80% (always-small) to 66% (+Router) on held-out, causing
-9pp task pass. Three competing explanations: (a) TF-IDF vocabulary mismatch on held-out
prompts, (b) sample-size insufficiency (178 training examples for TF-IDF+LR), (c) the
router was trained when large=GPT-5.4 was better than small, but small is now specialized
and dominates. EXP-066 (L2 regularization sweep) addresses (b).  
**Fix:** EXP-066 (blocked on A800). Short-term: add a 1-sentence explanation in §7 distinguishing these three causes.

### W3 — Full=Router in both 4-cycle HumanEval and MBPP ablation (IMPORTANT, NEW DIMENSION)
**Location:** Tables 4 and 9  
The main MBPP ablation (Table 4) shows Full=Router on routing metrics (+2pp code pass only).
The 4-cycle HumanEval (Table 9) shows Full=Router on ALL metrics including task pass.
The paper now explains the HumanEval case mechanistically (52.4% zero-variance). But the
MBPP ablation case (Table 4) lacks an analogous diagnosis — the +2pp is on a different
eval benchmark (MBPP) than the routing eval (HumanEval prompts). This asymmetry is
confusing for readers.  
**Fix:** Add a clarifying note in §6: the Full=Router routing result holds because routing quality depends on prompt-level decisions, not task accuracy; GRPO improves code accuracy without changing which prompts need large-model routing.

### W4 — Three disconnected experimental tracks (IMPORTANT, UNCHANGED)
**Location:** Sections 3–7  
Tracks A (HumanEval skills routing, GPT-5.4/DeepSeek), B (MBPP GRPO, Qwen2.5-Coder-1.5B),
C (tau2 joint evolution, Qwen3.6-35B-A3B/GPT-5.4), and now D (4-cycle HumanEval DAPO,
GPT-5.5/Qwen3.6-35B-A3B) use different benchmarks, models, and cost models.
An adversarial reviewer will argue this is four separate experiments, not a unified system.  
**Fix:** Add a paragraph in §1 explicitly framing Tracks A–D as ablation studies of MERA components on different modalities/scales, not as simultaneous deployments.

### W5 — No comparison to concurrent routing baselines on tau2 (IMPORTANT, UNCHANGED)
**Location:** Section 7, tau2 tables  
No evaluation of FrugalGPT-style cascade, random-50% baseline, or MoA on tau2 tasks.  
**Fix:** Add tau2 evaluation of a random-50% routing baseline (runnable in <1 hour on H200).

### W6 — Tau2 cycle instability (IMPORTANT, UNCHANGED)
**Location:** Tables 5–6  
Joint run: 78.38%→89.19%→70.27%→72.97%. Fullsplit: 70.22%→72.47%→70.22%→69.66%.
Neither shows monotonic improvement. With SE≈±5.3pp at 74 tasks, single-cycle comparisons
are unreliable.  
**Fix:** Add SE bars to tau2 cycle tables. Flag explicitly that cycle-to-cycle oscillation
is within ±1 SE of noise at this sample size.

### W7 — 52.4% zero-variance finding is cross-section (IMPORTANT, NEW)
**Location:** Sections 5.3 and 7  
The 52.4% zero-variance finding (§5.3) is from the 4-cycle HumanEval experiment, but
the failure mode analysis (§7) primarily discusses the MBPP setting (G=4, 73%). The paper
now correctly cross-references both (§7 now cites §5.3), but reviewers may ask: is the
52.4% DAPO result on HumanEval directly comparable to the 73% standard GRPO result on
MBPP? They use different models (Qwen3.6-35B-A3B vs Qwen2.5-Coder-1.5B), different
benchmarks, and different G values.  
**Fix:** EXP-081 (zero-variance scaling curve G=4/8/16/32 on the same model+benchmark)
would produce a clean apples-to-apples comparison. Pending A800. Short-term: add a
sentence noting that the two settings differ on model and benchmark, so the 73%→52.4%
gap reflects both G scaling and DAPO, not G alone.

### W8 — Implementation details incomplete (UNCHANGED)
**Location:** Sections 4–5  
BERT router: inference latency not reported. GRPO: KL penalty configuration (none?),
clip parameters (ε=0.2 from DAPO?), and whether no-KL mode is used — not stated.  
**Fix:** Add one sentence each: "BERT inference adds X ms/query on CPU." "We use no KL penalty, ε_low=0.2, ε_high=0.5 (DAPO)."

- Output: zero-variance fraction vs G curve (4 points) for the paper's failure mode section
