## Top 3 Relevant Papers

### 1. "GRPO-λ: Credit Assignment Improves LLM Reasoning" ★★★
**ArXiv ID**: 2510.00194 | 2025

**Core finding**: Vanilla GRPO assigns the same binary reward to every token in a sequence
(pass=+1, fail=−1), regardless of where the critical decision was made. GRPO-λ replaces
this with an eligibility-trace credit assignment: each token t receives a λ-discounted
sum of future rewards computed via the token-level log-probability sequence, i.e.
`credit_t = Σ_{k≥t} λ^(k-t) · r_k` where r_k is the per-token local reward signal
derived from the overall outcome. With λ=0.9, recent tokens near the test execution are
weighted 3–10× more heavily than opening tokens, reflecting the causal structure of
code generation (import statements contribute little; the core algorithm logic is decisive).
On code reasoning benchmarks, GRPO-λ achieves **+3 to +4.5 pp** over vanilla GRPO without
any change to data, model, or rollout count.

**Relevance**: Our +2 pp GRPO gain uses flat binary reward on all tokens in the response.
If credit is mis-assigned (boilerplate tokens pulling gradient away from the key logic),
the true useful gradient is diluted. GRPO-λ with λ=0.9 tests this hypothesis on the
1.5B model with the optimal recipe. Estimated wall-clock: ~90 min (identical to baseline
GRPO; λ-return computed post-hoc over already-generated rollouts with no extra LLM calls).

### 2. "Subspace Geometry Governs Catastrophic Forgetting in Low-Rank Adaptation" ★★★
**ArXiv ID**: 2603.02224 | February 2026

**Core finding**: Forgetting severity in LoRA sequential fine-tuning is governed by the
**minimum principal angle (MPA)** between task gradient subspaces. Specifically, when
consecutive task batches project into nearly parallel LoRA subspaces (small MPA), the new
task's gradient directly overwrites the previous task's solution in weight space, causing
severe forgetting. The paper proves this analytically and validates it empirically: a
simple MPA diagnostic computed from a small probe set (50 examples) correctly predicts
forgetting severity before training completes, enabling early stopping or regularisation
decisions.

**Relevance**: Our clearest unexplained observation is the 200→400 task degradation: 200
tasks gains +2 pp but 400 tasks LOSES 1 pp on MBPP eval100. Two hypotheses compete:
(H1) catastrophic forgetting of pre-trained knowledge (HumanEval retention drops); 
(H2) MBPP-specific overfitting / reward hacking (MBPP saturates but pre-training is fine).
The MPA diagnostic distinguishes these: if MPA collapses as training tasks accumulate from
200→400, H1 is confirmed; if MPA stays large but MBPP test-set variance shrinks, H2 is
confirmed. The diagnostic output directly informs the next regularisation target.

### 3. "Scaf-GRPO: Scaffolded Group Relative Policy Optimization" ★★
**ArXiv ID**: 2510.19807 | 2025

**Core finding**: In standard GRPO, when all G rollouts for a task either all-fail or
all-pass, the group advantage is exactly zero — training halts on that task ("learning
cliff"). Scaf-GRPO detects this collapse and injects minimal hints (retrieved from a
small scaffold bank) into the prompt for exactly those tasks, only for the current step,
so the model can generate at least one diverse rollout and resume gradient flow. On hard
coding benchmarks, Scaf-GRPO recovers 2–3 pp vs vanilla GRPO by keeping gradient signal
non-zero on tasks where the model is stuck.

**Relevance**: We observe ~53/200 all-fail tasks in our MBPP training set (26.5% zero
gradient). Scaf-GRPO targets exactly these stuck tasks. However, EXP-050 (rollout replay
difficulty selection) already addresses the same zero-advantage problem via a different
mechanism. Scaf-GRPO is noted but deferred to avoid queue bloat — if EXP-050 shows gains,
Scaf-GRPO becomes the natural follow-up.

### EXP-052: `exp_2026_06_05_001_grpo_lambda_credit_assignment_15b`

| Field | Value |
|-------|-------|
| Kind | `grpo_continual` |
| Priority | 9 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `credit_assignment="lambda_return"`, `lambda_decay=0.9` |
| Wall-clock est. | ~90 min (same as baseline GRPO) |
| GPU | auto (1× A800 80GB) |

**Rationale**: arxiv:2510.00194 (GRPO-λ, 2025) shows that vanilla GRPO's uniform binary
reward signal is sub-optimal: all tokens in a response receive identical credit regardless
of their causal role. Eligibility-trace credit assignment (λ-return with λ=0.9) weights
tokens by their temporal proximity to the outcome, concentrating gradient on the decisive
algorithm-implementation tokens vs boilerplate. The paper reports +3 to +4.5 pp over
vanilla GRPO on code reasoning benchmarks.

**Why this is not in the 51 already queued**: EXP-013 (DAPO), EXP-018 (F-GRPO), EXP-019
(MT-GRPO), EXP-050 (rollout replay), EXP-051 (adaptive budget) all modify task selection,
rollout generation, or advantage normalisation — none changes the per-token credit function
after rollouts are collected. This is the first experiment modifying within-sequence reward
attribution.

**Distinction from EXP-007 (token-level SFT reward weighting)**: EXP-007 reweights SFT
cross-entropy by token importance; GRPO-λ applies eligibility traces to RL advantage
estimates over binary executable rewards. Different training paradigm (GRPO vs SFT),
different reward source (executable test vs teacher distribution), different mechanism.

### EXP-053: `exp_2026_06_05_002_forgetting_eval_subspace_geometry_15b`

| Field | Value |
|-------|-------|
| Kind | `forgetting_eval` |
| Priority | 7 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 400 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | HumanEval eval100 @ checkpoints [100, 200, 400] tasks |
| Novel fields | `checkpoint_task_intervals=[100,200,400]`, `eval_at_checkpoints=True`, `eval_benchmark="humaneval"`, `subspace_angle_probe_size=50` |
| Wall-clock est. | ~60 min (eval phases fast; angle probe = 1 fwd-bwd pass over 50 examples) |
| GPU | auto (1× A800 80GB) |

**Rationale**: arxiv:2603.02224 (Subspace Geometry Forgetting, Feb 2026) proves that
forgetting in LoRA fine-tuning is governed by the minimum principal angle (MPA) between
task gradient subspaces: small MPA → high subspace overlap → new task overwrites previous
solution. Our history shows a clean cliff: 200 tasks → +2 pp, 400 tasks → −1 pp. Two
hypotheses compete: (H1) catastrophic forgetting (HumanEval drops as training accumulates);
(H2) MBPP-specific overfitting (HumanEval stable but MBPP eval subset saturates or reward
hacks). This forgetting_eval runs the standard 400-task GRPO recipe, evaluates HumanEval
at task-count checkpoints 100/200/400, and logs the MPA between consecutive task-batch
Jacobians. The diagnostic output is the prerequisite for all future continual-learning
mitigation experiments in the queue: if H1, EWC/LoRA-orthogonal regularisation; if H2,
reward-diversity or held-out test-set filtering.

**Why this is not already in the queue**: EXP-003 (forgetting_eval) from 2026-05-15
evaluates MBPP retention but uses the same benchmark for training and eval, making H1 vs H2
indistinguishable. This experiment uses **HumanEval** (never seen during MBPP GRPO training)
as the retention probe, which cleanly separates MBPP-specific forgetting from general code
capability degradation. Additionally, the MPA gradient subspace diagnostic is not present
in any prior forgetting_eval experiment.
