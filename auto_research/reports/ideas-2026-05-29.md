### 1. `arxiv:2605.11461` — Breaking Winner-Takes-All: Cooperative Policy Optimization Improves Diverse LLM Reasoning
*May 12, 2026 — Chen, Liang, Zheng, Hu (ISEE Lab, Sun Yat-sen University)*

**Highest-priority addition today: addresses the exploration collapse failure mode that
has capped all 35 queued experiments at +2pts.**

The paper identifies a root cause of premature convergence in GRPO that is distinct from
all 35 mechanisms already in the queue: **winner-takes-all collapse**. In standard GRPO with
binary reward and G=4 rollouts per task:

For a mixed-outcome task (1 pass / 3 fail), the model updates to increase probability of the
1 passing rollout. In subsequent training steps, the model generates the same solution more
frequently — so the "winning" rollout occupies 2 out of 4, then 3 out of 4. The group
advantage collapses toward zero as the winning solution dominates. The model has stopped
exploring alternative valid solutions; it has found one and is exploiting it exclusively.

This is a **within-task convergence problem** distinct from the **across-task saturation**
observed in our history:
- Across-task: we know 53/100 eval tasks are never solved (all-fail), 20/100 always solved
- Within-task (winner-takes-all): the 27/100 mixed tasks converge to a single solution mode
  rather than building a diverse internal model of all valid approaches

**GCPO mechanism: team-level valid-solution coverage credit**

```python
# For each group of G=4 rollouts on a task:
passing = [i for i in range(G) if reward[i] == 1.0]
# Deduplicate passing rollouts by normalized AST hash
unique_ast_hashes = set()
coverage_score = [0.0] * G
for i in passing:
    ast_hash = normalized_ast_hash(rollout[i])   # ast.parse + strip var names + sha256
    if ast_hash not in unique_ast_hashes:
        coverage_score[i] = 1.0                  # first unique passer: reward
        unique_ast_hashes.add(ast_hash)
    # else: duplicate passer → coverage_score[i] = 0 (no advantage bonus)

# Replace standard GRPO advantage with coverage-based advantage:
advantage = (coverage_score - mean(coverage_score)) / (std(coverage_score) + eps)
```

**Critical property for all-fail tasks:** when all 4 rollouts fail, `coverage_scores = [0,0,0,0]`,
`mean=0`, `std=0`, advantage = 0 — **identical to standard GRPO**. Zero regression risk on
the 53% all-fail tasks. The improvement is targeted at the 27% mixed-outcome tasks.

**Why this breaks the +2pt ceiling:** On a task where the model already knows one correct
solution, standard GRPO reinforces that solution until it dominates 3-4 rollouts, then
the group advantage collapses to near-zero and training stagnates. With GCPO, the first
rollout using solution A gets coverage credit; subsequent rollouts using the *same* solution A
get zero coverage credit (they're duplicates). This creates a persistent exploration incentive:
the model is rewarded for producing solution B (e.g., a different algorithmic approach) rather
than solution A again, even when A consistently passes. Over training, the model builds a
richer internal distribution over valid solutions for each task — which translates to higher
pass@1 because the model approaches each test-time question from multiple valid angles.

**Key results from paper:**
- GCPO vs. vanilla GRPO: +8-15% relative improvement on tasks with G_pass ≥ 2 (our
  mixed-outcome class)
- No regression on G_pass = 0 tasks (all-fail): identical to GRPO by construction
- Primary evaluation: Qwen2.5-7B on AIME2024, MATH500 (math reasoning); code-domain
  transfer expected because the AST-diversity mechanism is domain-agnostic

**Implementation overhead:** Python `ast.parse` + normalize identifiers + sha256 per rollout.
~0.4ms per rollout vs. ~4000ms forward pass = 0.01% overhead. ~60 additional lines.
**Estimated GPU time: ~92 minutes.**

### 2. `arxiv:2406.16768` — WARP: On the Benefits of Weight Averaged Rewarded Policies
*Rame et al., Google DeepMind, NeurIPS 2024*

**Zero-training 25-minute diagnostic: finds the optimal interpolation coefficient between
the existing GRPO adapter and the base model, potentially yielding a free +1pt improvement.**

WARP establishes that for any RL policy θ_RL trained on a reward r with implicit KL
regularisation budget β, there exists an optimal weight-averaged policy:

```
θ_WA(α) = α × θ_RL + (1 - α) × θ_base       α* ∈ (0, 1)
```

The theory shows α* = λ/(λ + β) where λ is the RL reward strength and β is the KL
coefficient. For our standard GRPO (no explicit KL, PPO clip ε=0.2), the implicit β is
difficult to estimate — but α* can be found by grid search.

**Why this might help:** Our 1.5B GRPO adapter (200×4, binary reward, no explicit KL) was
trained on 200 MBPP tasks with binary reward. With no explicit KL penalty, the adapter
may have drifted slightly beyond the optimal point on the reward-vs-generalization trade-off:
it may have increased pass rate on the 27 mixed-outcome training tasks by
overfitting patterns that occasionally fail at test time. Weight averaging at α=0.7 would
preserve 70% of the adapter's learned changes — the largest weight changes (genuine
improvements) survive, while smaller weight changes (overfitting artefacts) shrink toward
zero.

**For LoRA specifically:** Since the base model's LoRA weights are initialized to zero,
interpolation simplifies to: `lora_weights_WA = α × lora_weights_adapter`. This is a
trivial 1-line change: just scale the adapter's LoRA weight matrices by α before loading.
Five eval passes (α ∈ {0.3, 0.5, 0.7, 0.9, 1.0}) × ~5 minutes each = **~25 minutes total.**

**Three diagnostic outcomes:**
- α < 1.0 optimal (e.g., α=0.7 gives 50+/100): adapter overshoots; add `kl_coeff=0.02`
  to all future experiments as a standard practice, and apply WARP post-hoc as a free
  improvement to existing adapters
- α = 1.0 optimal (49/100 remains best): no overshoot; the +2pt ceiling is due to
  training signal quality, not regularisation, motivating REINFORCE++ EMA, EP-GRPO, etc.
- α = 0.3 optimal (severe overshoot): strong evidence that ALL future experiments need
  explicit KL penalty; the standard GRPO recipe is fundamentally unstabilized

**Distinct from all 35 existing experiments:** No prior experiment performs LoRA weight
interpolation. All eval experiments use the adapter at full α=1.0. Contrastive decoding
(EXP-035) amplifies adapter signal at inference time; WARP modifies the adapter weights
in parameter space — orthogonal mechanisms.

### 3. `arxiv:2510.04474` — DRPO: Efficient Reasoning via Decoupled Reward Policy Optimization
*Li, Chen, Lin, Yang — October 2025 (deferred)*

DRPO decouples the learning signal for correct vs. incorrect rollouts: correct rollouts are
normalized within the positive group only, shielded from interference by negative samples.
The key result is 77% **length reduction** with only 1.1% performance loss on GSM8k — DRPO
primarily targets verbosity bias rather than task accuracy.

**Why deferred:** Our training uses `max_new_tokens=192` (hard token cap), which already
prevents verbosity growth. The DRPO length-normalization mechanism solves a problem we don't
have at our sequence-length regime. The normalization improvement (decoupling positive/negative
groups) is partially covered by REINFORCE++ EMA (EXP-034, global EMA baseline already
addresses cross-group contamination). Revisit if max_new_tokens is relaxed beyond 384 or
if we observe systematic length increase in generated solutions.

lr=5e-6, LoRA r=16, binary reward, eval@100

**Rationale:** arxiv:2605.11461 (GCPO, Chen et al., May 2026) replaces GRPO's individual
rollout advantage with team-level valid-solution coverage credit: a rollout earns the coverage
bonus only if it passes AND its normalized AST is unique among passing rollouts in the group.
This prevents winner-takes-all convergence on the 27/100 mixed-outcome MBPP tasks — where
standard GRPO reinforces a single solution until the group advantage collapses — by keeping
a persistent exploration incentive: the model is rewarded for finding *novel* valid solutions
rather than repeating the one it already knows. On all-fail groups (our 53% majority), GCPO
reduces identically to standard GRPO (zero gradient). Targets the specific failure mode that
caps gains at +2pts across all 35 queued experiments. ~60 additional lines (ast.parse +
normalized deduplication + coverage advantage recompute). **Estimated: ~92 minutes.**

**Distinction from existing queue:**
- vs. F-GRPO (EXP-018): Task-level difficulty weighting vs. rollout-level diversity weighting. ✓
- vs. EP-GRPO (EXP-028): Token-level entropy gate vs. rollout-level AST uniqueness. ✓
