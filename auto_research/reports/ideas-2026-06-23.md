## Top 3 Relevant Papers

### 1. "VeRPO: Verifiable Dense Reward Policy Optimization for Code Generation" ★★★
**ArXiv ID**: [2601.03525](https://arxiv.org/abs/2601.03525) | January 7, 2026

**Core finding**: Standard GRPO/DAPO uses binary outcome reward (+1 all tests pass, 0 otherwise), discarding all signal from partially correct solutions. VeRPO constructs dense rewards from dynamically weighted partial test success:

1. **Dynamic test difficulty weights**: For each unit test `t` in problem `p`, compute `w(t) = 1 - mean_{rollouts}[pass(t)]` — the test's failure rate across all rollouts in the current training batch. Tests most rollouts fail get high weight; mastered tests (most rollouts pass) get low weight.
2. **Dense reward**: `R_dense(r) = Σ_t [w(t) × pass(r,t)] / Σ_t [w(t)]` — a weighted pass fraction emphasizing the hardest unsolved tests.
3. **Fusion**: `R_final = α × R_dense + (1-α) × R_outcome` where `R_outcome` is the standard binary DAPO signal. Paper recommends `α=0.5`.

**Why it fixes P-GRPO's failure (EXP-009)**: P-GRPO used a static fraction of passed tests as a single-component reward. As training progressed, the model passed most tests on easy problems, and the fractional reward converged toward 1.0, eliminating advantage variance. VeRPO prevents this: as the model masters easy tests, those tests' difficulty weights `w(t)` drop toward zero, and hard tests gain proportionally higher weight. Reward variance is maintained throughout training by continuously redirecting attention to unsolved tests.

**Why actioned as EXP-086 (priority 8)**:
- VeRPO is never mentioned in any prior report (confirmed via `grep -r "verpo\|2601.03525" auto_research/`).
- Directly addresses the core limitation of binary reward: 52.4% of DAPO groups have zero variance (all-pass or all-fail). While RL-ZVP (EXP-082) handles zero-variance groups via entropy-scaled advantage, VeRPO reduces zero-variance incidence by making the reward more granular — a group where one rollout passes 7/8 tests and another passes 5/8 tests is non-zero-variance under VeRPO but zero-variance under binary DAPO.
- EXP-009's failure provides a clear mechanistic prediction: VeRPO should succeed where P-GRPO failed because P-GRPO's pathology (static weights + converging reward → collapsed variance) is directly addressed by VeRPO's dynamic reweighting.
- Implementation ~30 lines in `grpo_train_simple.py`. Wall-clock ~90 min.

**Duplicate check**: EXP-009 (P-GRPO) = static fraction, failed. VeRPO = dynamic per-test weighting + binary fusion. EXP-085 (GDPO) = multi-component independent normalization (different axis). No queued experiment has `verpo` or `dynamic_testweight`. ✓

### 2. "FOREVER: Forgetting Curve-Inspired Memory Replay for Language Model Continual Learning" ★★★
**ArXiv ID**: [2601.03938](https://arxiv.org/abs/2601.03938) | January 2026

**Core finding**: LLM forgetting, when measured against optimizer update magnitude (cumulative L2 norm of parameter changes), follows the Ebbinghaus forgetting curve — rapid forgetting immediately after large parameter updates, decaying exponentially as the model stabilizes. FOREVER replaces fixed-step replay intervals with a forgetting-curve-aligned schedule:

1. **Model time τ**: Defined as `Σ ||Δθ||_2` (cumulative optimizer update norm), not raw training steps. A large gradient step advances τ more than a small step, reflecting actual forgetting risk.
2. **Ebbinghaus-scheduled replay**: The next replay event triggers when predicted forgetting `(1 - e^{-τ/τ_0}) > θ_forget`. After large updates, τ grows fast → replay is triggered soon. After the model stabilizes, τ grows slowly → replay intervals lengthen.
3. **Intensity-aware regularization**: Replay loss is scaled by the predicted forgetting severity at current model time, giving stronger anti-forgetting gradient when forgetting is predicted to be high.

Tested on 0.6B–13B decoder-only LLMs across three continual learning benchmarks. Consistently reduces forgetting while maintaining forward transfer (plasticity).

**Why actioned as EXP-087 (priority 7)**:
- Never mentioned in any prior report (confirmed via `grep -r "forever\|2601.03938" auto_research/`).
- Direct application to W6: The tau2 cycle decline (70.22%→72.47%→70.22%→69.66%) is the one open AAAI paper weakness with no mitigation. FOREVER is a principled mitigation with strong theoretical grounding and empirical validation.
- **Complementary to EXP-084** (SFT-only diagnostic, queued June 22): EXP-084 diagnoses the CAUSE (is it SFT or GRPO?); EXP-087 tests a MITIGATION (FOREVER replay). If both experiments run, the AAAI paper can close W6 with both diagnosis and remedy.
- Applicable to tau2 SFT Phase 3a: the `tau2_train_wrapper.sh` SFT loop performs LoRA parameter updates, where `||Δθ||_2` is computable. Previous-cycle SFT training pairs are already stored (`cycle_{n-1}/sft_data.jsonl`). The episodic buffer (50 samples) requires minimal memory.

**Duplicate check**: EXP-079 (on-policy GRPO replay, HumanEval): replays GRPO rollouts within GRPO phase — different phase, data, scheduling. FOREVER replays supervised SFT pairs within SFT phase via Ebbinghaus scheduling. No overlap. ✓
