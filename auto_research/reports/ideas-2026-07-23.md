## Top 3 Relevant Papers

### 1. arxiv:2605.27765 — Restoring the Sweet Spot: Pass-Rate Weighted Self-Distillation for LLM Reasoning
**Zehao Liu, Yuanpu Cao, Jinghui Chen, Vasant G. Honavar — Penn State — May 27, 2026**

**Core insight:** GRPO's group-relative advantage implicitly weights tasks by `p(1-p)` — the mathematical "sweet spot" that concentrates gradient signal on intermediate-difficulty tasks. Standard SFT/self-distillation weights all tasks equally, so easy tasks (p≈1, flat demonstrations) and hard tasks (p≈0, zero positive signal) crowd out the informative intermediate tasks. SC-SDPO restores the sweet spot by explicitly weighting each task's SFT loss by `ŵ = √[p̂(1−p̂)]`:

```
L_SC_SDPO = Σ_task √[p̂(task)(1 − p̂(task))] × L_SFT(task)
```

where `p̂` is the empirical small-model pass rate from training traces.

**Key results:**
- +3.2 pass@16, +4.3 maj@16 on Qwen3-8B (scientific reasoning benchmarks)
- +1.8 pass@16, +3.0 maj@16 on OLMo-3-7B
- Stable training dynamics (no entropy collapse or loss spikes)

**Why not already in queue:** EXP-121 (ZPD-SFT) uses a binary threshold to FILTER tasks outside [0.2, 0.8] pass rate. SC-SDPO uses CONTINUOUS √(p(1-p)) weights with no threshold — different mechanism, complementary to ZPD-SFT (could be combined). No prior queue entry addresses Phase 3a SFT via weighted loss.

**Direct relevance:** Our skills arm at 75.61% vs large 96.34% is 20.7pp behind (Problem C). Phase 3a SFT is currently uniform-weighted. With SCALING_FORCE_BOTH=1 we already have per-task small-model pass rates in traces.jsonl — the weights are free to compute.

### 2. arxiv:2605.17106 — HyDRA: Hybrid Dynamic Routing Architecture for Heterogeneous LLM Pools
**June 12, 2026**

**Core method:** Replaces binary-logit routing classifiers with a ModernBERT encoder producing K=4 independent sigmoid heads — each head scores a query along one capability dimension (reasoning, code generation, debugging, tool use). Model selection uses "shortfall matching": select the cheapest model whose per-dimension capability profile meets or exceeds all predicted scores.

**Key results:**
- 86 ms median CPU inference latency in production (viable for online routing)
- Fully decoupled from model catalog: adding/removing models requires only a config change, no router re-training
- Outperforms single-logit routers on heterogeneous task alignment benchmarks
- Better calibration across difficulty axes than single-binary-output classifiers

**Why novel vs queue:** All 127 prior queue experiments use logistic regression (or threshold-based heuristics) over TF-IDF or handcrafted features. HyDRA is the ONLY queued experiment proposing a pre-trained language model encoder for routing. Multi-head sigmoid output is unique — it decomposes routing decisions into interpretable sub-axes that can be individually ablated and reported.

**Direct relevance:** Our router accuracy is 92.68% (logistic regression, raw prompt). The gap to large (96.34%) is 3.66pp — small but real. HyDRA's ModernBERT encoder captures code structure, algorithmic patterns, and contextual difficulty that TF-IDF misses. More importantly, multi-head output provides per-axis routing interpretability for the AAAI paper (which dimensions drive routing decisions?), strengthening the paper's analysis section. All CLAUDE.md design constraints are maintained: router trains on raw prompt only, no procedure prefix.

### 3. arxiv:2607.16850 — Group Entropy-Controlled Policy Optimization (GEPO)
**Guangran Cheng et al. — July 18, 2026 (added to queue as EXP-126 on 2026-07-21)**

Already queued. Included here as context for the July 2026 GRPO literature snapshot:

GEPO addresses entropy-dependent bias in GRPO's group-relative normalization: easy-task groups (low entropy, all rollouts succeed) and hard-task groups (high entropy, all fail) produce z-score magnitudes that are statistically non-comparable, allowing easy-task gradients to crowd out hard-task learning. GEPO applies entropy-conditioned asymmetric attenuation using the 25th/75th percentile of group entropies per mini-batch as self-calibrating thresholds — no hyperparameter tuning needed.
