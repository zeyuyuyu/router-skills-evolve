## Top 3 Relevant Papers

### 1. SRPO: Unifying Group-Relative and Self-Distillation Policy Optimization via Sample Routing ★★★
**ArXiv ID:** 2604.02288 | April 2026

**Core finding:** SRPO splits GRPO rollouts by outcome: passing rollouts → standard GRPO advantage-weighted reinforcement; failing rollouts → SDPO logit-level correction toward π_ref. The SDPO loss pushes the policy back toward the reference distribution at token positions where it has diverged on incorrect outputs:

```
L_SDPO(failed_rollout) = mean_t [ max(0, log π_ref(r_t|ctx) - log π_θ(r_t|ctx) - margin) ]
L_total = L_GRPO(passed) + λ_sdpo * L_SDPO(failed)
```

This converts zero-gradient all-fail groups into active training signal. All required tensors (ref_logp, pol_logp) are already computed by DAPO's existing KL penalty term — no extra forward passes.

**Results:** +3.2% pass@1 on HumanEval-Instruct (CodeQwen 1.5B), +4.7% on MBPP-plus vs standard DAPO.

**Relevance:** Our 43/82 all-fail groups (52.4% ACR) currently waste all backward-pass compute with zero gradient. SRPO converts each of the 344 wasted failed rollouts into a SDPO correction signal. Unlike all queued ACR experiments (which modify reward functions within the GRPO path), SRPO adds a new loss head for the failed path — orthogonal and potentially combinable with STARE/AVSPO/SA-AH-GRPO. Motivated **EXP-106**.

### 2. DemoPSD: Disagreement-Modulated Policy Self-Distillation ★★★
**ArXiv ID:** 2607.02502 | July 2, 2026 (2 days ago) | Yunhe Li, Hao Shi, Wenhao Liu et al.

**Core finding:** Standard on-policy self-distillation (OPSD) distills from the teacher uniformly across all tasks, including easy ones where the student already succeeds. This causes:
- Overfitting to in-domain easy-task patterns
- Suppression of exploration on hard tasks
- Privileged information leakage (student learns answer-dependent shortcuts)

DemoPSD's fix: only distill when student and teacher *disagree* (teacher correct, student wrong). On agreement tasks (both correct), apply a small weight λ=0.1 to preserve numerical stability while letting disagreement-task gradient dominate.

**Results:** +2.1% across diverse reasoning benchmarks vs. uniform OPSD; better out-of-domain generalization on held-out tasks.

**Relevance:** Our SFT pipeline (Phase 3a) trains on all large-model traces where SCALING_FORCE_BOTH=1 collected them — including Category B tasks (both models succeed). These B-category traces crowd the SFT gradient on skills the small model already has. DemoPSD's disagreement filter directly targets our 20.7pp skills-vs-large gap by focusing SFT on hard tasks where only the large model succeeds. Pure data-weighting change (~15 lines in `traces_to_sft.py`). Motivated **EXP-107**.

### 3. When Do Intrinsic Rewards Work for Code Reasoning? A Comprehensive Study ★★ (Diagnostic)
**ArXiv ID:** 2606.20881 | June 2026 | Purdue / UC Berkeley / UC Santa Barbara

**Core finding:** Certainty-based intrinsic reward methods (entropy of model logits, token-level uncertainty) yield early-cycle gains on code reasoning but **inevitably collapse**: models progressively shorten outputs and lose reasoning capability. Collapse speed depends on (a) sample size and (b) training temperature.

**Relevance:** Several queued ACR experiments use entropy-based token-level signals: STARE (EXP-090), SA-AH-GRPO (EXP-098), EP-GRPO (EXP-102). This paper warns that these methods may show pass@1 gains in cycle 1 but degrade in cycles 2–3. **Action item (not a new experiment):** add output-length tracking and cross-cycle reasoning retention metrics when these experiments run. If any queued entropy-based ACR experiment shows output shortening, do not report its cycle-1 gain as a final result.

### EXP-106: SRPO Sample-Routed Policy Optimization for All-Fail Groups (HumanEval)
**ID:** `exp_2026_07_04_001_srpo_sample_routing_allfail_grpo_humaneval`
**Kind:** `grpo_continual` | **Priority:** 8 | **Estimated GPU:** 1.75h

**Motivation:** arxiv:2604.02288. All queued ACR approaches (EXP-090/094/096/098/101/102/104) modify the reward function or loss weighting *within* the GRPO path to reduce how many groups collapse. SRPO takes a different approach: leave the GRPO path unchanged, but route all-fail rollouts to a complementary SDPO loss head that provides reference-anchored logit correction. This is architecturally orthogonal — it converts wasted compute (344 all-fail rollouts per cycle) into genuine training signal without modifying rewards or adding new rollouts.

**Spec:** 1-cycle HumanEval, Qwen2.5-Coder-1.5B, G=8, SCALING_FORCE_BOTH=1. Add SDPO loss: at each token in a failed rollout, penalize divergence from π_ref beyond a margin. Combined loss: `L = L_DAPO(passed) + λ_sdpo * L_SDPO(failed)`. Three variants: λ_sdpo ∈ {0.1, 0.5, 1.0} plus DAPO baseline. All variants share the same rollouts (collect once, reuse). ~20 lines additional code using already-computed ref_logp tensors.

**Target:** pass@1 ≥ 93.68% (+1pp from 92.68%) in best variant. If achieved: Full > Router for the first time → AAAI headline result. SDPO active fraction should be ~52.4% (all-fail group fraction), confirming wasted compute is recovered. Output length stable (SRPO anchors to π_ref, preventing entropy collapse).
