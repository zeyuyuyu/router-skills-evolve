## Top 3 Relevant Papers

### 1. "ReSkill: Reconciling Skill Creation with Policy Optimization in Agentic RL" (★★★)
**ArXiv ID**: 2606.01619 | He, Lin, Han, Zhu, Fang, Wang, Zhu, Li, Reimherr | Submitted June 1, 2026

**Core finding**: Existing skill-augmented RL methods decouple skill creation from policy
optimization, causing skills to drift out of alignment with the evolving policy. ReSkill
introduces an RL-in-the-loop framework that exploits GRPO's group-wise rollout structure
for joint skill-policy learning. Key mechanisms:
1. **Assertion-driven skill creator**: after every K gradient steps, analyzes failing rollouts
   from mixed-outcome tasks, extracts recurring assertion failure patterns, generates
   conditional trigger-based skills (e.g., "When processing sequences, check boundary cases").
2. **Within-group skill comparison**: splits the G-rollout GRPO group — half uses the current
   skill library, half does not — allowing gradient-following skill selection rather than
   offline heuristics.

**Relevance**: Our project is named `router-skills-evolve` — the joint evolution of skills
alongside routing is its core thesis. Our 27/100 mixed-outcome MBPP tasks (which likely share
recurring failure modes: boundary conditions, off-by-one, empty container handling) are the
ideal testbed. A persistent, task-agnostic skill library extracted from these failures could
unlock cross-task improvements that single-task hint injection (Scaf-GRPO) misses.

---

### 2. "DRA-GRPO: Your GRPO Needs to Know Diverse Reasoning Paths" (★★★)
**ArXiv ID**: 2505.09655 | Huang et al. | May 2025

**Core finding**: Standard GRPO has a Diversity-Quality Inconsistency — distinct reasoning
paths receive identical scalar rewards, causing policy collapse into dominant solution modes.
DRA-GRPO proposes Diversity-aware Reward Adjustment (DRA) using Submodular Mutual Information
(SMI) of sentence embeddings to calibrate rewards within each rollout group. A semantically
unique rollout receives a boosted effective reward; redundant rollouts are downweighted. The
SMI marginal gain ensures diminishing returns (first unique rollout gets the largest boost).

**Results**: 58.2% average accuracy on 5 math benchmarks with Qwen-1.5B-Distill, 7,000
samples. Plug-and-play: integrates with any GRPO variant without changing architecture.

**Relevance**: Our G=4 rollouts on 27 mixed-outcome tasks almost certainly collapse to 2-3
semantically identical solutions. DRA-SMI would directly reward the rare diverse rollout,
addressing the +2pp plateau via semantic diversity incentivization. Mechanistically distinct
from all 47 queued experiments: GCPO (AST hash), VPO (per-test-case coverage), unlikeliness
(token log-probability) — DRA uses full-rollout semantic embeddings.

---

### 3. "Towards Efficient Multi-LLM Inference: Characterization and Analysis of LLM Routing and Hierarchical Techniques" (★★)
**ArXiv ID**: 2506.06579 | Behera, Champati, Morabito, Tarkoma, Gross | June 2025

**Core finding**: Systematic empirical comparison of routing (single model selection per
query) vs. cascading/hierarchical inference (sequential escalation). For code tasks with
bimodal difficulty, confidence-based cascade routing often matches or exceeds learned-router
performance at lower training cost. The paper also identifies that routing gains diminish
rapidly when small-model capability is weak (below a "competence threshold").

**Relevance**: Confirms findings from our own e2e ablation. Supports existing routing
experiments (EXP-017, EXP-022, EXP-034) but adds the competence-threshold insight:
our small model at 49/100 is near the lower bound of useful routing. Improving the small
model's pass rate (via GRPO) is more impactful than tuning the router threshold — reinforces
the training-first priority of the current queue. No new experiment proposed based on this
paper (routing track already well-covered by 5 queue entries).

## Experiments Queued Today

### EXP-048: `exp_2026_06_03_001_dra_grpo_smi_diversity_15b`

| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 8 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `dra_reward_adjustment=True`, `dra_encoder="all-MiniLM-L6-v2"`, `dra_lambda=0.5` |
| Wall-clock est. | ~95 min |
| GPU | auto (1× A800 80GB) |

**Rationale**: DRA-GRPO (arxiv:2505.09655) identifies that standard GRPO's scalar binary
reward causes Diversity-Quality Inconsistency: G=4 rollouts for mixed-outcome tasks collapse
to semantically identical dominant solutions. DRA uses Submodular Mutual Information of
sentence embeddings to compute a marginal coverage gain for each rollout:
`gain_i = (G-1 - Σ_{j≠i} sim(i,j)) / (G-1)`, then scales the reward:
`effective_reward_i = reward_i × (1 + 0.5 × gain_i)`. The frozen all-MiniLM-L6-v2 encoder
(22M params, <0.5GB VRAM, 0.2s per batch of 4) adds negligible overhead.

**Distinction from existing queue**: GCPO (AST structural hash), VPO (per-test-case
coverage vector), unlikeliness reward (token log-probability rarity under base model),
entropy-gated GRPO (token distribution entropy) — DRA uses global **semantic embedding
cosine similarity**, a fully orthogonal diversity axis. The only experiment targeting
semantic-meaning distinctiveness rather than structural, functional, or token-level diversity.

### EXP-049: `exp_2026_06_03_002_reskill_skill_library_grpo_15b`

| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 8 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `algorithm="reskill"`, `skill_update_freq=15`, `skill_max_library_size=8`, `skill_revision_model="self"` |
| Wall-clock est. | ~100 min |
| GPU | auto (1× A800 80GB) |

**Rationale**: ReSkill (arxiv:2606.01619, submitted June 1, 2026) reconciles skill creation
with policy optimization by exploiting GRPO's group-wise structure. Every 15 gradient steps,
the runner analyzes failing rollouts from mixed-outcome tasks, extracts common assertion
failure patterns (IndexError, wrong operator, boundary conditions), and uses the model itself
in inference mode to condense these into ≤8 conditional skills injected into the system
prompt. Within-group comparison (G/2 rollouts with skills, G/2 without) lets the gradient
validate each skill revision. Skills that consistently improve rollout success rate over 3
cycles are retained in the persistent library. Our project is named `router-skills-evolve`
— this is the first experiment directly testing the "skills-evolve" thesis: a dynamic,
task-agnostic skill library co-trained with the GRPO policy.

**Distinction from existing queue**: Scaf-GRPO (EXP-023, per-task static hints triggered by
stagnation) and MURPHY (EXP-030, task-local multi-turn execution feedback) both operate
per-task and ephemerally. ReSkill builds a **persistent, task-agnostic skill library**
updated throughout training via gradient-following skill selection. No prior experiment
tests joint skill-library + policy co-training. The self-revision inference step (~8s per
cycle × 13 cycles ≈ 104s total) is negligible vs. training time.
