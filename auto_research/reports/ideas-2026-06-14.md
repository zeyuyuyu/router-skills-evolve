## Top 3 Relevant Papers

### 1. "EP-GRPO: Entropy-Progress Aligned Group Relative Policy Optimization with Implicit Process Guidance" ★★★
**ArXiv ID**: 2605.04960 | May 6, 2026

**Core finding**: Standard GRPO has three credit assignment failures at the token level:

1. **Uniform token granularity**: All tokens in a completion receive the same scalar advantage, whether they represent a high-stakes branch point (`if`, `return`, `raise`) or a deterministic low-information token (whitespace, closing brackets). This wastes gradient capacity on tokens that don't need updating.

2. **Uniform polarity**: The group-relative normalizer can flip advantage sign for individual tokens within a mixed completion. A correct reasoning step inside a failed completion gets penalized; an incorrect step inside a passing completion gets rewarded. This is especially harmful when G=4, as the outcome of 3/4 rollouts dominates the sign.

3. **Zero-variance collapse**: When all G rollouts have identical outcomes (all-pass or all-fail), the group-relative standard deviation is zero, producing zero advantage for all tokens — the gradient signal vanishes entirely for those prompts. With our 200-task training set and G=4, this affects ~50-73% of prompts at any given point in training.

**Fixes**:
- **Entropy-gated advantage**: Each token's advantage is scaled by its entropy (computed from the log-prob distribution over the vocabulary). High-entropy tokens (genuine branching choices) receive amplified advantage; low-entropy deterministic tokens are down-weighted.
- **Implicit process signal**: The log-prob ratio between the current policy and reference model (`log π_θ(a_t) - log π_ref(a_t)`) is used as a dense, self-supervised process reward. It propagates directional guidance between outcomes even for zero-variance groups.

**Results**: Qwen2.5-3B, 5 math reasoning datasets: GRPO 18.14% → EP-GRPO 22.93% (+4.79 pp). Qwen2.5-7B: similar gains. The mechanism is domain-agnostic; no results on MBPP/code generation yet — that's what EXP-068 tests.

**Why actioned today (EXP-068)**: The zero-variance collapse (failure mode 3) is our leading hypothesis for the 400-task degradation. EP-GRPO addresses it without changing the training loop structure, reward, or LoRA config — only the advantage computation changes. Implementation cost: token entropy is computable from existing generation log-probs; policy/reference ratio is already computed for the KL penalty. Zero additional inference cost.

**Distinct from existing queue**: EXP-052 (GRPO-λ) uses temporal discounting by position. GRPO-S (arxiv:2508.04349) was deferred pending GRPO-λ results and uses simpler entropy term without process signal. EP-GRPO is newer (May 2026) and the combined entropy-gating + process signal is sufficiently distinct to queue independently.

### 2. "CLPO: Curriculum Learning meets Policy Optimization for LLM Reasoning" ★★★
**ArXiv ID**: 2509.25004 | Updated June 8, 2026

**Core finding**: Static easy-to-hard curricula (like EXP-062/063) have a fundamental limitation: the difficulty proxy is computed once using the *base* model's rollout pass rates, then held fixed throughout training. As the model improves, tasks that were initially "medium difficulty" become easy and produce all-pass groups with zero gradient signal — but the static schedule continues sampling them at high frequency.

CLPO (Curriculum Learning meets Policy Optimization) addresses this by:
1. Using **on-policy rollout accuracy** from the *current* model (not the base model) as the difficulty proxy.
2. Updating this proxy every K gradient steps, dynamically reclassifying tasks as the model's capability evolves.
3. **Simplifying hard problems** that the model consistently fails into sub-tasks, maintaining gradient signal at the upper difficulty tail.
4. **Diversifying medium problems** (those the model solves with ~30-60% rollout rate) by generating paraphrased variants, increasing data coverage at the current capability frontier.

**Why actioned today (EXP-069)**: EXP-062 uses a static `base_pass_at_4` proxy and a linear difficulty schedule. CLPO's online re-estimation adds ~2 rollouts per task every 20 steps (small overhead) but ensures the difficulty proxy tracks the actual training dynamics. The key question is whether this online tracking is worth the overhead — if EXP-069 beats EXP-062 by ≥0.5pp, online estimation becomes the default for all future curriculum experiments.

**Distinct from existing queue**: EXP-062 (`curriculum_strategy="difficulty_sorted_easy_to_hard"`) uses static base_pass_at_4. EXP-063 is the 400-task extension. EXP-065 is the reverse (hard-to-easy) ablation. EXP-069 (`curriculum_strategy="online_clpo"`) is the first experiment using current-policy rollouts for dynamic re-estimation.

### 3. "AEM: Adaptive Entropy Modulation for Multi-Turn Agentic Reinforcement Learning" ★★
**ArXiv ID**: 2605.00425 | May 1, 2026

**Core finding**: In multi-turn agentic RL (where reward is only available at the end of a long trajectory), sparse outcome rewards create a severe credit assignment problem. AEM addresses this by elevating entropy analysis from token level to **response level** (each complete action in a multi-turn trajectory) and adaptively modulating the exploration-exploitation trade-off based on per-action entropy.

**Why relevant but not actioned today**: EXP-067 (tau2 agentic GRPO, queued 2026-06-12-v2) is already in queue. AEM would be a natural augmentation of EXP-067 *if* EXP-067 returns a non-zero Method A pass rate. Deferred pending EXP-067 result.
