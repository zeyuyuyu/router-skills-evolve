## Top 3 Relevant Papers

### 1. VIMPO: Value-Implicit Policy Optimization for LLMs
**arxiv:2606.20008** · Kang et al., UC Berkeley · June 18, 2026

**What it does:** Derives a value function implicitly from the KL-regularized RL
optimality conditions, representing V(s_t) as the suffix log-ratio of current vs
reference policy over remaining tokens. Provides denser, per-token advantage
estimation without any learned critic. In an autoregressive generation MDP:

```
V_implicit(s_t, r) = Σ_{τ=t..T} [log π(a_τ|s_τ) - log π_ref(a_τ|s_τ)]
A_VIMPO(t, r) = (V_implicit(s_t, r) - mean_r V_implicit(s_t, r)) / std_r + R_r
```

**Why relevant:** Our 52.4% zero-variance group discard rate is the primary GRPO
bottleneck. VIMPO eliminates this by construction: even when all rollouts have the
same final reward (e.g., all pass), the suffix log-ratios differ across rollouts,
so per-position variance is non-zero. DAPO discards entire groups; VIMPO discards
nothing while still weighting tokens by their actual policy divergence from reference.

**Implementation cost:** ~40 lines in `grpo_train_simple.py`; logprobs are already
computed per token in the current rollout collection step.

**Distinct from prior queue:** VeRPO (EXP-086) changes the REWARD value; VIMPO
changes how ADVANTAGES are estimated from a given reward. RL-ZVP (EXP-082) penalizes
zero-variance post-hoc; VIMPO prevents it structurally.

### 2. UCCI: Calibrated Uncertainty for Cost-Optimal LLM Cascade Routing
**arxiv:2605.18796** · Anon., May 11, 2026

**What it does:** Two-step upgrade to any existing LLM cascade router:
1. **Isotonic regression calibration**: maps raw classifier confidence P̂ to a true
   error probability ê = g(P̂), achieving O(n^{-1/3}) ECE convergence.
2. **Cost-constrained threshold**: selects τ* = argmin cost(τ) s.t. error_rate(τ) ≤ δ.
   Under this calibrated score, τ* is provably cost-optimal (Theorem 2 in paper).

**Result:** On 75k NER queries with 4B/12B LLM cascade — 31% inference cost reduction
vs entropy thresholding and FrugalGPT-style learned thresholds. ECE: 0.12→0.03.

**Why relevant:** Our router in `train_router_simple.py` uses raw scikit-learn
probabilities with a heuristic threshold. At N~82 training traces, expected ECE is
0.10-0.15 — the classifier is overconfident. UCCI calibration is orthogonal to
everything else (doesn't change features, doesn't change base model) and is compatible
with design decision #3 (raw prompt features, no procedure prefix). If our current
27.56% large-model cost follows the same pattern as UCCI's 31% reduction, we could
push toward ~20% large-model cost while maintaining 92%+ task pass.

**Implementation cost:** ~20 lines in `train_router_simple.py` post-training step.

### 3. Curriculum Reinforcement Learning from Easy to Hard Tasks Improves LLM Reasoning
**arxiv:2506.06632** · June 2026

**What it does:** Progressive difficulty sampling for GRPO — starts with problems
where the current model has P(solve) ∈ [0.3, 0.7] (the "learning zone"), gradually
expanding to harder problems as performance improves. Uses online difficulty estimation
from recent rollout pass rates. Shows +3-5% on MATH vs uniform GRPO.

**Why relevant:** A complementary angle to VeRPO and VIMPO for addressing the
zero-variance collapse. Problems where all rollouts pass (P≈1.0) or all fail (P≈0.0)
contribute zero-variance groups. Curriculum sampling focuses the GRPO budget on
problems in the high-variance regime, implicitly reducing group discard rate.
However, this paper was already partially addressed by EXP-068/EXP-069 (curriculum
with HumanEval difficulty tiers). We note it here as context but do not queue a new
experiment today — the VIMPO approach is more principled and novel for our setup.
