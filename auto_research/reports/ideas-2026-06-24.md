## Top 3 Relevant Papers

### 1. "Beyond GRPO and On-Policy Distillation: An Empirical Sparse-to-Dense Reward Principle for Language-Model Post-Training" ★★★
**ArXiv ID**: [2605.12483](https://arxiv.org/abs/2605.12483) | May 2026 | Xu et al.

**Core finding**: Sparse sequence-level reward (GRPO with binary pass/fail) and dense token-level teacher supervision (on-policy distillation, OPD) are not competing methods — they occupy different reward-density regimes suited to different model scales. The paper establishes the following empirical principle:

> **Sparse RL** (GRPO) is most useful for **strong models** that can explore and discover better behavior from noisy terminal rewards.
> **Dense distillation** (OPD, forward-KL) is better for **compressing** discovered behavior into a small student.

Experimental validation on Qwen3 and Llama models, verifiable math tasks:
- For a fixed **Qwen3-1.7B** deployment student: distilling from a GRPO-improved 8B teacher outperforms applying GRPO directly to the 1.7B with the same labeled-data budget.
- A **forward-KL warmup** on teacher rollouts (5 steps) followed by **OPD** on student rollouts outperforms standalone student GRPO at all checkpoints measured.

**Why actionable as EXP-088** (priority 8):
- Our Phase 3b runs binary GRPO on **Qwen2.5-Coder-1.5B** — exactly the "small student, sparse reward" setting the paper identifies as sub-optimal.
- We already have GPT-5.5 traces from Phase 1 (teacher rollouts); OPD does not require new API calls.
- Implementation: replace binary reward accumulation in `grpo_train_simple.py` with token-level forward-KL toward GPT-5.5 log-probs, with a 5-step forward-KL warmup. ~40 lines.
- Direct comparison to DAPO reference (EXP cycle 3): same data budget, same compute, different objective.
- If confirmed: paper AAAI 2027 claim becomes "for 1.5B code models, dense distillation outperforms sparse GRPO on the same teacher traces", with a direct ablation. Strong contribution.

### 2. "Reinforcement-aware Knowledge Distillation for LLM Reasoning" ★★★
**ArXiv ID**: [2602.22495](https://arxiv.org/abs/2602.22495) | April 2026 | Zhang et al. (Adobe Research)

**Core problem**: Standard `KL(student || teacher)` added as a regularizer to GRPO suffers from **objective interference**:
1. **Distribution mismatch**: the teacher's token distribution is computed under the teacher's policy, not the student's evolving policy mid-training.
2. **Competitive objectives**: teacher imitation may pull the student toward teacher-style solutions on problems the student already solves, degrading reward maximization.

**RLAD mechanism** — selective, advantage-gated teacher imitation:
```
L_RLAD = alpha * E_prompt [ max(0, A_teacher - A_student) * KL(student || teacher) ]
```
Where:
- `A_teacher(tau_T)` = advantage of the teacher rollout under the student's value baseline
- `A_student(tau_S)` = student's own rollout advantage
- `max(0, ...)` gate: skip teacher imitation when the teacher's rollout is NOT better than the student's current policy

Result: denser supervision signal on hard problems where the teacher outperforms the student, zero interference on easy problems where the student already matches or beats the teacher.

**Why actionable as EXP-089** (priority 7):
- We have GPT-5.5 traces from Phase 1 (teacher rollouts with binary pass/fail rewards), required for `A_teacher` computation.
- Zero-variance problem: 52.4% of DAPO groups have zero reward variance (all-pass or all-fail). On all-fail groups, `A_teacher > A_student = 0`; RLAD would add teacher KL to those groups, providing gradient signal that DAPO drops entirely.
- Implementation: look up cached GPT-5.5 trace per problem, compute `A_teacher = gpt55_reward - group_mean`, apply gate, add `alpha * KL` to GRPO loss. ~35 lines in `grpo_train_simple.py`.
- Complementary to EXP-088 (OPD): OPD replaces the GRPO objective; RLAD augments it. Together they bracket the design space for teacher-assisted Phase 3b.

### EXP-089: `exp_2026_06_24_002_rlad_selective_teacher_kl_grpo_humaneval`

| Field | Value |
|-------|-------|
| Kind | `grpo_continual` |
| Priority | 7 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Bench | humaneval |
| ArXiv ref | **2602.22495** |
| Estimated hours | 1.5h on A800 |

**Design**: Augment DAPO GRPO with a selective, advantage-gated teacher KL term (RLAD). For each problem in a GRPO batch:
- Compute `A_teacher = gpt55_reward(problem) - group_mean_reward`
- Compute `A_student = student_rollout_reward - group_mean_reward`
- If `A_teacher > A_student`: add `alpha=0.3 * KL(student || gpt55)` to GRPO loss
- If `A_teacher <= A_student`: skip (student already matches/beats teacher on this problem)

**Spec excerpt**:
```python
{
  "grpo_algo": "dapo_rlad",
  "rlad_alpha": 0.3,
  "rlad_gate": "advantage_weighted",
  "rlad_teacher": "openai/gpt-5.5",
  "rlad_teacher_trace_source": "phase1_collected_traces",
}
```

**Rationale summary**: arxiv:2602.22495 (RLAD) shows that standard KL regularization in GRPO competes with reward maximization due to distribution mismatch when teacher/student policies diverge. The advantage-gated selective imitation resolves this: teacher guidance is applied only where the teacher is demonstrably better (A_teacher > A_student). For our system: 52.4% of DAPO groups have zero reward variance (all-fail); on those groups, A_teacher > A_student = 0, so RLAD provides gradient signal DAPO completely drops. Expected effect: reduce zero-variance waste and improve hard-problem pass rate.
