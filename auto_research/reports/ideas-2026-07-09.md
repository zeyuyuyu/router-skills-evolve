## Today's Hot arxiv Tags

From web search (2026-07-09), most relevant papers to our project themes:

### Top papers found relevant

**1. arxiv:2507.20673 — Geometric-Mean Policy Optimization (GMPO)**
*(Zhao et al., Peking University / Tsinghua, 2025 — in meta.json since 07-03 as finding,
now promoted to explicit experiment)*

GRPO maximises the **arithmetic mean** of token-level importance ratios × advantage.
Rare tokens with high policy shift (r_t >> 1) dominate the sum → explosive gradients.
GMPO replaces with **geometric mean** via log-space aggregation:

```
L_GMPO = exp( mean_t( log(r_t) × A ) )
```

This bounds outlier token contributions logarithmically while leaving the clipping
unchanged. Plug-and-play: ~5-line change.

Empirical: GMPO-7B beats GRPO-7B by avg **+4.1pp** on six math benchmarks (AIME24,
AMC, MATH500, OlympiadBench, Minerva, Geometry3K) and **+1.4pp** on multimodal.
Code: github.com/callsys/GMPO.

Relevance: GMPO fixes token-level instability in the **47.6% non-degenerate groups**
that Sign advantage (EXP-108) leaves intact at the group level. The two mechanisms
are orthogonal → composable.

**2. arxiv:2605.28860 — Mechanistic Origins of Catastrophic Forgetting: Why Does RL
Preserve Circuits Better than SFT?**
*(Rojas, Sawant, Allen et al., May 28 2026)*

Circuit-level analysis on Qwen2.5-3B-Instruct: introduces *differential circuit
vulnerability* (DCV) to compare SFT vs RL fine-tuning across tasks.

Key finding: **SFT adapts faster** to the new task but **disrupts more circuits**
(higher forgetting). **RL (GRPO) preserves circuits** — updates are implicitly
conservative in parameter subspaces sensitive to prior tasks.

Mechanism: RL's conservatism is naturally scaled by reward variance (consistent with
arxiv:2507.05386's theoretical proof that RFT gradient ∝ reward variance). This is
an *implicit EWC* effect without explicit regularisation.

Relevance: explains the **tau2 oscillation** (89.19% → 70.27%) as SFT catastrophic
forgetting. Suggests replacing SFT in cycles ≥1 with GRPO-only to preserve circuits.

**3. arxiv:2604.19485 — EVPO: Explained Variance Policy Optimization**
*(April 2026 — not pursued today, noted for future consideration)*

Adapts between PPO (critic-based) and GRPO (batch-mean) at each training step by
monitoring batch-level explained variance. When EV > 0, critic reduces variance (use
PPO); when EV ≤ 0, critic inflates variance (use GRPO). Consistently outperforms both
fixed PPO and fixed GRPO across control / agentic / math tasks.

Not queued today: PPO requires a value head (extra compute, extra code). GMPO (EXP-110)
and Sign advantage (EXP-108) are simpler and more directly validated. EVPO is a
candidate for EXP-112 if EXP-110 and EXP-108 both fail.

### EXP-110: GMPO + Sign Combined for HumanEval GRPO
*(pending_queue_update_2026_07_09.py, priority=7, kind=grpo_continual)*

**Rationale:** arxiv:2507.20673 (GMPO).

Our DAPO pipeline has **two distinct gradient instability sources**:

| Level | Source | Fraction affected | Current fix |
|---|---|---|---|
| Group | Zero within-group variance → A=0 | 52.4% | EXP-108 Sign (A=2r-1) |
| Token | Outlier r_t >> 1 dominates arithmetic mean | All non-degenerate groups | **None (this EXP)** |

GMPO addresses the token level. Sign addresses the group level. Together:

```python
# Sign: group-level (EXP-108 formula, replaces group-mean centering)
advantages = 2.0 * rewards - 1.0

# GMPO: token-level (geometric mean via log-space)
log_r = logprobs - old_logprobs
clipped_log_r = log_r.clamp(-clip_eps, clip_eps)
loss = -torch.exp((clipped_log_r * advantages.unsqueeze(-1)).mean(-1))
```

Total change: ~7 lines in `grpo_train_simple.py`.

**Three variants (single run):**
- (A) GMPO only (standard group advantage, geometric token mean)
- (B) Sign + GMPO combined — **main variant**
- (C) DAPO baseline (reproduce ACR=52.4%, pass@1=92.68%)

**Target:** Variant B pass@1 > 93.68% (+1pp from DAPO baseline) for AAAI headline:
*Full system > Router for the first time.*

### EXP-111: RL vs SFT Forgetting Eval — HumanEval Continual (2 cycles)
*(pending_queue_update_2026_07_09.py, priority=6, kind=forgetting_eval)*

**Rationale:** arxiv:2605.28860 (RL preserves circuits better than SFT).

**The tau2 oscillation explained:**
Our tau2 uses SFT (FSDP2) per cycle. arxiv:2605.28860 shows SFT overwrites prior-task
circuits during each new fine-tuning round → catastrophic forgetting. The 89.19% →
70.27% drop is this effect made visible across cycles.

**Proposed experiment:**
Use `forgetting_eval` kind on HumanEval (local, fast) to compare two strategies over 2 cycles:

| Strategy | Cycle 0 | Cycle 1 |
|---|---|---|
| A (current default) | base → SFT → GRPO | GRPO → new SFT → new GRPO |
| B (GRPO-continual) | base → SFT → GRPO | GRPO → GRPO only (no new SFT) |

Hold out 20 HumanEval problems (D_held, ~12%) from all SFT batches. Measure:
- `pass@1(D_held, cycle k)` after each cycle
- `forgetting@1 = pass@1(D_held, cycle 1) − pass@1(D_held, cycle 0)`

**Prediction:** Strategy B (RL-only cycle 1) shows less negative forgetting@1 because
GRPO's conservative updates preserve D_held circuits from cycle 0.

**AAAI payoff:**
- B < A forgetting: "Drop SFT from cycles ≥1; use GRPO-continual." Fix for tau2 cycle 2+.
- B > A forgetting: SFT faster adaptation wins; informs how much SFT is needed.
- Either way: mechanistic diagnosis of tau2 oscillation, grounded in arxiv:2605.28860.

**Estimated GPU hours:** 3.5h (2 strategies × 2 cycles × ~0.9h each)
