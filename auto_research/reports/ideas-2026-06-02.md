## Top 3 Relevant Papers

### 1. "Critique-GRPO: Advancing LLM Reasoning with Natural Language and Numerical Feedback" (★★★)
**ArXiv ID**: 2506.03106 | Qin et al. | June 2026

**Core finding**: Standard GRPO's purely scalar (binary) reward creates a learning dead-end for
failed rollouts — there is no gradient signal on *why* the code failed. Critique-GRPO adds a
second stage: for each failed rollout, the model generates a natural-language critique of its
own error, then produces a refined attempt. Both initial and refined responses contribute to
the GRPO loss. A shaping function amplifies learning from correct-but-unfamiliar refinements:

```
eff_reward_refined = binary + lambda * (1 - exp(mean_log_prob_under_base / L))
```

Results: +15.0–21.6% pass@1 on Qwen-series models and +7.3% on Llama-3.2-3B across 8
reasoning tasks including HumanEval and MBPP variants. The critique quality of the 1.5B
model is modest but sufficient to provide directional error signal for simple MBPP failures
(wrong loop bounds, off-by-one, incorrect type conversion).

**Relevance**: Our 27/100 mixed-outcome MBPP tasks have rollouts that fail silently — the
model gets zero gradient and learns nothing from the failure mode. Critique-GRPO converts
these silent failures into a structured learning signal without an external oracle.
Mechanistically distinct from SCAF-GRPO (EXP-018: static problem hints) and MURPHY
(EXP-029: external verifier feedback between turns).

### 2. "Learning the Mechanism of Catastrophic Forgetting: A Perspective from Gradient Similarity" (★★)
**ArXiv ID**: 2605.09608 | May 10, 2026

**Core finding**: Catastrophic forgetting in LLM continual training is predicted by the cosine
similarity between the new-task gradient and the exponential moving average (EMA) of all
prior task gradients. When cosine_sim < 0.4, attention heads in lower layers show 15–23%
disruption (measured via representational similarity analysis). When cosine_sim > 0.4,
training on the new task neither helps nor hurts prior performance. The paper proposes a
simple gating rule: skip the gradient update for any mini-batch where cosine_sim < threshold.
Evaluated on continual SFT; the authors suggest the mechanism applies to continual RL.

**Relevance**: This is the most actionable explanation for our 200→400 task degradation.
Among the 200 MBPP training tasks, roughly 50 tasks (estimated) likely have gradients that
conflict with the beneficial direction established by the remaining 150. At 400 tasks, the
proportion of conflicting tasks is higher, driving the observed regression. Gradient cosine
gating would automatically exclude conflicting tasks at each step without any pre-labelling.
This experiment was deferred on 2026-05-31 pending C-CHAIN/KL results; since those
experiments are still queued (A800 offline 19 days), waiting longer is unjustified.

### 3. "Posterior-GRPO: Rewarding Reasoning Processes in Code Generation" (★)
**ArXiv ID**: 2508.05170 | August 2025 (noted for future consideration)

**Core finding**: Binary test-pass reward treats all passing code equally regardless of the
intermediate reasoning quality. Posterior-GRPO adds a per-step reward for correct intermediate
reasoning: code comments that accurately describe what the next code block does receive a small
bonus (+0.05 per correct comment). A 7B model achieves +4.5% pass@1 over binary-reward GRPO
baseline on HumanEval.

**Relevance**: Potentially applicable after Critique-GRPO (EXP-048) establishes whether
multi-stage GRPO helps. Not queued today because:
(a) requires a comment-correctness oracle (LLM judge or AST match) adding ~30% overhead;
(b) the 1.5B model generates few substantive comments by default (would need prompt engineering
    to elicit them);
(c) at 49 pending experiments, queuing process supervision now would push the backlog further.
Deferred until after A800 comes back online and at least 10 experiments from the backlog run.

### EXP-048: `exp_2026_06_02_001_critique_grpo_selfcrit_15b`
| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 8 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 initial + up to 4 critique-refined per failed rollout |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `critique_grpo=true`, `critique_type="self_critique"`, `refinement_rollouts=4`, `refinement_shaping_lambda=0.3` |
| Wall-clock est. | ~110 min (1.7× standard GRPO; critique gen + refinement for ~50% failed rollouts) |
| GPU | auto (1× A800 80GB) |

**Rationale**: Critique-GRPO (arxiv:2506.03106, June 2026) shows that augmenting GRPO with a
natural-language self-critique stage for failed rollouts delivers +15–21.6% pass@1 on Qwen
models. Our 27/100 mixed-outcome training tasks produce ~50% failed rollouts that carry zero
gradient under standard GRPO (rank bias: all-passing or all-failing groups have trivial
advantages). The critique-refinement stage converts failed rollouts into a gradient-contributing
learning signal: the model generates NL critique ("the loop bound is exclusive not inclusive"),
then produces a refined code solution; both responses contribute to the GRPO loss with a
shaping function that up-weights rare-but-correct refinements. This is mechanistically distinct
from all 47 queued experiments: SCAF-GRPO (EXP-018) injects static hints at generation time
(no gradient from any refinement pass); MURPHY (EXP-029) uses external verifier feedback
(not the model's own NL critique); Unlikeliness (EXP-046) re-weights initial rollout
advantages only (single-stage, no refinement).

### EXP-049: `exp_2026_06_02_002_grad_cosine_gate_15b`
| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 7 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `grad_cosine_gate=true`, `grad_cosine_threshold=0.4`, `grad_ema_beta=0.95` |
| Wall-clock est. | ~95 min (same as best case; gate adds ~2 ms/step overhead over LoRA params) |
| GPU | auto (1× A800 80GB) |

**Rationale**: The 200→400 task degradation (47→46 vs 47→49) is the most critical unexplained
failure in the project. arxiv:2605.09608 (May 2026) provides a mechanistic explanation:
new-task gradients that conflict (cosine_sim < 0.4 with the running gradient EMA) cause
lower-layer attention disruption that correlates directly with forgetting. Gradient cosine
gating — skip any GRPO gradient update where cosine(g_step, g_ema) < 0.4 — would
automatically exclude conflicting tasks at each step without static pre-labelling or proxy
tasks. This addresses the core question of whether our +2 pp gain is bottlenecked by a
minority of gradient-conflicting tasks, and is orthogonal to all existing regularization
approaches in the queue: KL-penalty (EXP-002/042) applies distributional regularization
globally; C-CHAIN (EXP-027) reduces step budget for recent tasks; FRPO (EXP-018-004)
applies a max-min KL-ball formulation. Gradient cosine gating is the only parameter-space,
per-step, per-task filtering mechanism in the 49-experiment queue. Previously deferred on
2026-05-31 pending C-CHAIN/KL results — with A800 offline for 19 days and no results
expected soon, waiting further is unjustified.
