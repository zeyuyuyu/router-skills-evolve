## Top 3 Relevant Papers

### 1. "Addressing Performance Saturation for LLM RL via Precise Entropy Curve Control" ★★★
**ArXiv ID**: 2604.26326 | April 2026

**Core finding**: Standard GRPO and most RLVR algorithms suffer from performance
saturation because their entropy collapses in an uncontrolled way. Once the policy
concentrates on too few output paths, rollout diversity drops to near-zero and gradient
signal vanishes — the model stops improving even with continued training. This is a
global training-dynamics problem distinct from the per-prompt zero-variance group
collapse addressed by EP-GRPO (EXP-068).

The paper introduces **Entrocraft**: a rejection-sampling approach that controls the
empirical entropy curve by filtering completions whose per-step entropy falls outside
a user-specified tolerance band around a target schedule. Completions are generated
normally but only gradients from on-schedule completions are applied. The implementation
cost is minimal: entropy is computable from the existing logit outputs; rejected
completions cost generation time but not backprop time.

A systematic sweep of entropy schedule shapes reveals:
- **Linear annealing** (high start → lower end) outperforms all alternatives
- Flat schedules, sudden drops, and cosine annealing all underperform linear
- The key insight: models need exploration early (high entropy) and consolidation
  late (lower entropy, but not collapsed) — linear annealing naturally delivers this

**Results**:
- Sustains improvement for up to **4× longer** before saturation vs. standard GRPO
- **pass@K diversity** raised by **+50%**
- A **4B model** with Entrocraft surpasses an **8B baseline** using standard GRPO

**Why actioned today (EXP-070)**: Our best result (47→49 MBPP, +2pp) has been
unchanged across all configurations at 200 tasks. The global entropy collapse
hypothesis predicts that the policy is saturating in the early phases of 200-task
training — Entrocraft's linear annealing directly tests whether controlled entropy
allows the +2pp to grow to +4pp or beyond. No change to reward, task count, LoRA
config, or learning rate — only the advantage sampling distribution changes.

**Distinct from existing queue**: EP-GRPO (EXP-068) uses per-token entropy to
re-weight advantage magnitudes within a completion — it does not control the global
entropy trajectory and does not use rejection sampling. No queued experiment uses
`entropy_control="entrocraft_rejection"` or `entropy_schedule="linear_annealing"`.

### 2. "GTPO: Stabilizing Group Relative Policy Optimization via Gradient and Entropy Control" ★★★
**ArXiv ID**: 2508.03772 | August 2025 (revised May 2026)

**Core finding**: Two structural instabilities in GRPO that are distinct from zero-
variance collapse and global entropy collapse:

1. **Conflicting gradient updates on shared tokens**: A token appearing in both a
   passing and a failing completion within the same group G receives both positive
   (from the passing completion) and negative (from the failing one) gradient signals.
   The net update depends on advantage magnitudes and can reduce the probability of
   a token the model has already learned is useful — an anti-gradient signal. This
   is especially damaging for high-value code tokens like `return`, `def`, `if`,
   `raise`, which frequently appear in both passing and failing completions.

2. **Destabilization from negatively-rewarded trajectories**: GRPO penalizes tokens
   in failing completions by pushing the policy away from the sampled token toward
   the broad vocabulary distribution. This is diffuse and destabilizing; it can
   increase entropy in ways that corrupt previously learned patterns.

**GTPO's fixes**:
- **Positive-only gradient for shared tokens**: When a token appears in both passing
  and failing completions in the same group, the negative gradient update from the
  failing completion is skipped. The positive gradient from the passing completion is
  amplified proportionally.
- **Trajectory-level entropy filter**: Completions whose trajectory-level entropy
  exceeds a provable threshold (derived from group advantage statistics) are filtered
  out entirely before gradient computation.
- **No reference model / No KL divergence**: Stability guarantees come from the
  gradient and entropy filters rather than KL regularization. This removes the
  reference model forward pass (~10-15% wall-clock saving) and eliminates the KL
  coefficient as a confound.

**Results**: GSM8K, MATH, AIME 2024/2025, AMC 2023 (math reasoning). Strong
across all benchmarks. No code generation results yet — that's what EXP-071 tests.

**Why actioned today (EXP-071)**: The conflicting gradient on shared tokens is a
plausible mechanism for our observed degradation: as GRPO trains on 200 MBPP tasks
with G=4, the model's most-used syntax tokens (`def`, `return`, `if`, `for`) appear
in nearly every completion — both passing and failing. GTPO's skip-negative filter
would protect these tokens from anti-gradient noise while still allowing learning
from outcome-differentiated completions. The no-reference-model variant also serves
as an ablation on whether our KL coefficient (default typically 0.01-0.05) is a
confound in the +2pp ceiling.

**Distinct from existing queue**: EP-GRPO (EXP-068) uses token-level entropy gating
(re-weighting advantage magnitude by per-token entropy) and retains the reference
model with KL regularization. GRPO-S (2508.04349, deferred) uses sequence-level
entropy bonus — additive to loss, not a gradient filter, retains reference model.
No queued experiment has `kl_coeff=0.0` or `use_reference_model=False`.

### 3. "On Entropy Control in LLM-RL Algorithms" ★★
**ArXiv ID**: 2509.03493 | September 2025

**Core finding**: Conventional entropy regularization (adding an entropy bonus to
the GRPO loss) gives weak or zero gains in LLM-RL because the LLM's response space
is enormous and the optimal outputs are sparse. Standard entropy bonuses spread
probability mass across the entire vocabulary, not over useful alternatives. The
paper's AEnt method addresses this by evaluating entropy on a re-normalized policy
restricted to a smaller "candidate token space" (tokens with non-negligible probability
under the current policy), combined with an auto-tuned clamping coefficient.

**Why relevant but not actioned today**: Entrocraft (EXP-070, arxiv:2604.26326)
is a more recent paper (April 2026 vs. September 2025) that addresses the same
saturation problem via rejection sampling rather than an auxiliary bonus term.
Entrocraft has stronger empirical results (4× longer training, 4B beats 8B) and
requires no auxiliary coefficient tuning. AEnt is deferred pending EXP-070 result:
if Entrocraft fails to break the +2pp ceiling, AEnt's re-normalized candidate-space
entropy bonus provides a distinct follow-up mechanism.
