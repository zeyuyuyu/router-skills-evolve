## Top 3 Relevant Papers

### 1. "Subspace Geometry Governs Catastrophic Forgetting in Low-Rank Adaptation" ★★★
**ArXiv ID**: 2603.02224 | February 2026
*(Previously identified in project hotspot 2026-06-05 at ★★★; used as rationale for EXP-053)*

**Core finding**: Forgetting severity in LoRA sequential fine-tuning is governed by the
**minimum principal angle (MPA)** between consecutive task gradient subspaces. When
task gradients project into nearly-parallel LoRA subspaces (small MPA), the new task's
update directly overwrites the previous task's solution in weight space. The MPA is
computable from a 50-example probe set before training completes, enabling early
diagnosis.

**Today's relevance** (beyond EXP-053): The MPA analysis has a direct implication for
LoRA rank choice that was not actioned in EXP-053. For a fixed dataset, lower LoRA rank r
spans a smaller parameter subspace. Two task gradient subspaces of dimension r drawn from
a high-dimensional weight space will, on average, have a larger minimum principal angle
than two subspaces of dimension 2r — the geometry of random subspace intersections favors
higher angles when dimensionality is lower relative to ambient dimension. This suggests
r=8 may be a simple, zero-overhead intervention to widen the MPA and reduce forgetting
at the 200→400 task boundary.

### 2. "CapTrack: Capacity-Tracking LoRA for Continual Fine-Tuning" ★★★
**ArXiv ID**: 2603.06610 | February 2026
*(Previously identified in project hotspot 2026-06-05 at ★★★; not yet actioned)*

**Core finding**: CapTrack augments LoRA training with a capacity-tracking penalty that
prevents the current task's LoRA column space from aligning with the previous task's
subspace. The penalty term is `L_orth = λ · ||A^T · A_prev||_F`, where A is the current
task's LoRA matrix and A_prev is the stored column space from the previous checkpoint.
This enforces gradient-level orthogonality between consecutive tasks without requiring
a separate task memory buffer or frozen model copy beyond the lightweight A_prev store.

**Results** (from project's prior hotspot entry): Significantly reduces forgetting on
sequential fine-tuning benchmarks while matching or exceeding single-task LoRA accuracy.
Overhead is approximately one extra matrix multiplication per gradient step.

**Relevance**: This is the direct algorithmic fix for the mechanism that EXP-053 is
measuring. If EXP-053 returns H1 (catastrophic forgetting via subspace collapse), CapTrack
is the prescribed intervention. Queuing it now means it is ready to run immediately once
EXP-053 completes its diagnosis, without waiting for another daily cycle.

## Experiments Queued Today

### EXP-056: `exp_2026_06_07_001_lora_rank8_400tasks_cliff_test`

| Field | Value |
|-------|-------|
| Kind | `grpo_continual` |
| Priority | 7 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 400 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / **8** (reduced from baseline r=16) |
| Eval | MBPP eval100 |
| Novel fields | `lora_r=8` (only change vs baseline 400-task run) |
| Wall-clock est. | ~95 min (r=8 → fewer params → faster steps than r=16) |
| GPU | auto (1× A800 80GB) |

**Rationale**: arxiv:2603.02224 (Subspace Geometry, cited in EXP-053) proves that
catastrophic forgetting in sequential LoRA fine-tuning is governed by the minimum
principal angle (MPA) between consecutive task gradient subspaces — lower MPA means
higher subspace overlap and more severe forgetting. All 55 experiments in the current
queue use LoRA r=16. The known 200→400-task degradation (47 → 46 MBPP eval100) is
consistent with the subspace-collapse mechanism: as more task batches accumulate, their
r=16 subspaces compete for the same directions in weight space.

Reducing r from 16 to 8 halves the subspace dimensionality. For a fixed ambient weight
space dimension (Qwen2.5-1.5B has ~72M adapter-eligible parameters), two rank-8 subspaces
have geometrically more room to remain orthogonal than two rank-16 subspaces — the
expected MPA between two random r-dimensional subspaces in an n-dimensional space
increases as r decreases. If this geometric argument holds empirically, r=8 will reduce
or eliminate the −1pp degradation at 400 tasks without requiring any new implementation
(only a single integer argument change).

**Why this is not in the 55 already queued**: Every prior `grpo_continual` experiment
uses `lora_r=16`. No rank-ablation experiment exists in the queue or in history.
EXP-053 (forgetting_eval) probes the MPA diagnostic but does not change the rank.
EXP-056 tests the rank as the independent variable.

**Expected outcome**: If r=8 at 400 tasks achieves ≥47/100 (no degradation below
baseline), lower rank is confirmed as a forgetting mitigation strategy and all subsequent
400+ task experiments should adopt r=8. If r=8 degrades at 200 tasks too (loss of
capacity), this tells us the +2pp gain requires r≥16 capacity and rank reduction is
not viable.

### EXP-057: `exp_2026_06_07_002_captrack_subspace_orthogonal_400tasks`

| Field | Value |
|-------|-------|
| Kind | `grpo_continual` |
| Priority | 6 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 400 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `lora_regularization="captrack"`, `captrack_orthogonality_coeff=0.01`, `captrack_task_boundary_interval=50` |
| Wall-clock est. | ~115 min (orthogonality penalty adds one matmul per gradient step) |
| GPU | auto (1× A800 80GB) |

**Rationale**: arxiv:2603.06610 (CapTrack, Feb 2026; identified in project hotspot
2026-06-05 at ★★★ alongside arxiv:2603.02224) provides the direct algorithmic
intervention for the forgetting mechanism that EXP-053 is measuring. CapTrack adds
a Frobenius-norm penalty `L_orth = λ · ||A^T A_prev||_F` to the training loss at
each gradient step, where A is the current LoRA A-matrix (or its column space, stored
per-task) and A_prev is the column space from the previous task checkpoint. This
penalty directly penalizes the optimizer from rotating A into the subspace already
occupied by A_prev, enforcing gradient-level orthogonality without a separate memory
replay buffer.

EXP-057 is the complement to EXP-056: EXP-056 tests whether reducing rank (r=8)
is sufficient to widen the MPA without an explicit penalty; EXP-057 tests whether an
explicit orthogonality penalty at r=16 achieves the same forgetting mitigation with
full rank capacity. The two are not mutually exclusive — if both work, a follow-up
could combine r=8 with CapTrack penalty for maximal subspace separation.

`captrack_task_boundary_interval=50` means A_prev is updated every 50 MBPP task
boundaries (approximately every 50 tasks through the 400-task training run), giving
8 penalty update points. `captrack_orthogonality_coeff=0.01` is the λ value from
the CapTrack paper's recommended configuration for code/math LoRA fine-tuning.

**Why this is not in the 55 already queued**: No prior experiment uses
`lora_regularization` or any subspace-level orthogonality constraint. EXP-002 (KL
penalty) adds a distribution-level KL divergence to prevent policy drift but does not
address gradient subspace alignment between task batches. EWC-style experiments were
deferred in earlier planning; CapTrack is the 2026 state-of-the-art version of the
same class of ideas and is directly motivated by the MPA theory in arxiv:2603.02224.
