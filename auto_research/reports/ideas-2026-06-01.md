## Top 3 Relevant Papers

### 1. "Rewarding the Unlikely: Lifting GRPO Beyond Distribution Sharpening" (★★★)
**ArXiv ID**: 2506.02355 | He, Fried, Welleck (CMU) | June 2025

**Core finding**: GRPO has a *rank bias* — within a group of G rollouts, all passing rollouts receive equal positive advantage regardless of how rare their token sequences are under the base model. This causes distribution sharpening: the model learns to reproduce its already-dominant solution patterns rather than explore novel-but-correct ones. When plotted, pass@k curves for GRPO models saturate earlier than sampling-only models from the base distribution.

**Fix**: *Unlikeliness reward* — add a bonus `lambda * (1 - exp(per_token_log_prob_under_base))` to the binary reward before group normalisation. Rare-but-correct rollouts receive higher effective reward than common-but-correct rollouts. Result: +3.1% pass@1 on Lean4 miniF2F without changing any other hyperparameter.

**Relevance**: Our 27/100 mixed-outcome MBPP tasks produce passes almost certainly biased toward the model's dominant algorithmic priors. The unlikeliness bonus would incentivise discovery of alternative correct solutions — directly addressing the +2pp saturation plateau that has persisted across all 45 prior experiments.

---

### 2. "Improving LLM-Generated Code Quality with GRPO" (★★)
**ArXiv ID**: 2506.02211 | Robeyns, Aitchison (Bristol) | June 2025

**Core finding**: Binary test-pass reward cannot distinguish between maintainable and brittle passing code. Adding a composite static-analysis quality score (cyclomatic complexity, line count, flake8 errors) to GRPO reward improves solution maintainability by 15–22% on HumanEval without reducing pass@1. Blinded human annotators rate quality-GRPO outputs as significantly more maintainable.

**Relevance**: Our current 49/100 passing solutions may include verbose or hacky code that passes tests via brute force. Quality-GRPO would strengthen the paper's contribution: not just correct, but maintainable code from the small model.

---

### 3. "Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey" (★★)
**ArXiv ID**: 2603.04445 | March 2026

**Core finding**: Survey of routing paradigms — identifies that for code tasks with *bimodal difficulty* (easy for small model or impossible regardless of scale), confidence-based cascade routing often dominates learned-classifier routing. Our BERT-based learned router achieves 93% accuracy; the survey suggests this may plateau without difficulty-signal calibration.

**Relevance**: Supports the existing router threshold experiments in queue (EXP-017, EXP-022); also motivates the gold-label eval (EXP-041). No new experiments added today based on this paper — routing track is already well-covered in queue.

### EXP-046: `exp_2026_06_01_001_unlikeliness_reward_grpo_15b`
| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 8 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Train tasks | 200 (MBPP aug, excl. eval20) |
| Rollouts | 4 |
| LR / LoRA r | 5e-6 / 16 |
| Eval | MBPP eval100 |
| Novel fields | `reward_bonus="unlikeliness"`, `unlikeliness_lambda=0.5` |
| Wall-clock est. | ~95 min |
| GPU | auto (1× A800 80GB) |

**Rationale (condensed)**: GRPO has rank bias (arxiv:2506.02355) — all passing rollouts receive equal advantage regardless of base-model probability. For our 27/100 mixed-outcome tasks, this means the model reinforces its already-dominant solution patterns and never learns rare-but-correct alternatives. Adding `effective_reward = binary + 0.5 * (1 - exp(mean_log_prob_base / L))` before advantage computation incentivises rare-trajectory learning. Requires a frozen base-model forward pass on each rollout (~5% overhead; already supported by KL experiments). Distinct from all 45 queued experiments: F-GRPO weights by task-level success rate (not rollout log-prob), GCPO rewards AST-structural uniqueness (not token-probability rarity), POPO eliminates negative groups (not a reward signal), VPO decomposes by test case (orthogonal).
