### 1. `arxiv:2605.06650` — Beyond Negative Rollouts: Positive-Only Policy Optimization with Implicit Negative Gradients (POPO)
*May 7, 2026 — Mingwei Xu, Hao Fang*

**EXP-040 target. The only new RLVR algorithm paper in the search window that introduces a
mechanistically distinct training paradigm not yet covered by any of the 43 queued experiments.**

The core observation: in standard GRPO, explicit negative-reward rollouts are required to form
a non-trivial advantage estimate. When G=4 rollouts all fail (our 53/100 all-fail training tasks),
all rollouts have identical reward (r=0), group mean = 0, group std = 0, advantage = 0 — zero
gradient. POPO resolves this by proving that the negative-reward rollouts are not needed at all:
bounded importance sampling of the positive-rollout distribution implicitly encodes the negative
gradient via probability mass redistribution.

**POPO advantage computation:**

```python
# For G=4 rollouts where at least 1 passes:
rewards = [r0, r1, r2, r3]  # binary {0, 1}
if max(rewards) == 0:
    # all-fail group: skip — no positive rollout, no gradient this step
    continue

# positive-rollout IS weight:
pos_Z = sum(rewards) + eps
for i, r_i in enumerate(rewards):
    A_i = r_i / pos_Z  # IS weight: passing rollout gets 1/n_passers, failing gets 0
    A_i = clip(A_i * IS_correction, 0, 1 / (1 - popo_is_clip))

# Policy update: maximise P(positive rollout) using IS-bounded gradient
```

**Why this is distinct from EXP-039 (Unlearnability filter):**
EXP-039 performs a one-time pre-exploration phase (run 8 rollouts per task at temp=0.9 before
training), marks tasks with 0/8 partial passes as unlearnable, and excludes them permanently.
POPO is dynamic: at each gradient step, whichever tasks happen to produce ≥1 positive rollout
(under the CURRENT policy) contribute to training. As the policy improves, previously all-fail
tasks may spontaneously produce a positive rollout and become POPO-active — no pre-exploration
needed, no static exclusion, no missed opportunities.

**Why this is distinct from DAPO (EXP-013):**
DAPO dynamically resamples groups until EVERY group has ≥1 pass AND ≥1 fail. This requires
guaranteed mixed-outcome groups — impossible for truly hard tasks. POPO trains on
positive-only groups with implicit IS negatives, never requiring an explicit negative rollout.

**Estimated GPU time: ~85 minutes.** ~30 lines of change.

### 2. `arxiv:2605.22817` + `arxiv:2605.11461` — VPO + GCPO Orthogonal Diversity Composition
*EXP-041: First diversity-mechanism composition experiment in the queue.*

EXP-038 (VPO) and EXP-036 (GCPO) are both queued but not yet run. The composition is a
high-information experiment because the two mechanisms address orthogonal blind spots:

| Property | VPO | GCPO |
|----------|-----|------|
| **Signal** | Which test cases are passing | Which algorithmic structures are generated |
| **Diversity target** | Test-coverage patterns | Solution code structure |
| **Reward axis** | {0,1}^T per-test vector | AST-normalized uniqueness per rollout |
| **Blind spot** | Two AST-identical rollouts passing different tests = VPO-diverse, GCPO-same | Two AST-unique rollouts passing the same tests = GCPO-diverse, VPO-same |

The composition addresses both blind spots simultaneously:

```python
# Step 1: VPO vector advantage (per-test-case)
for t in range(T):
    mean_t = mean(r[:, t])
    std_t = std(r[:, t]) + eps
advantage_vpo = (1/T) * sum_t[(r[:, t] - mean_t) / std_t]   # shape: (G,)

# Step 2: GCPO AST uniqueness bonus
ast_hashes = [sha256(normalize_ast(code_i)).hexdigest() for i in range(G)]
unique_passers = set()
coverage_bonus = zeros(G)
for i in argsort(rewards)[::-1]:   # process in decreasing reward order
    if rewards[i] > 0 and ast_hashes[i] not in unique_passers:
        coverage_bonus[i] = gcpo_alpha / (len(unique_passers) + 1)
        unique_passers.add(ast_hashes[i])

# Final advantage: additive composition
advantage = advantage_vpo + coverage_bonus
```

Three informative outcomes regardless of direction: (a) additive → combine all future diversity
experiments; (b) bottleneck → one signal is redundant, guides simplification; (c) conflicting
→ gcpo_alpha should be reduced, the two axes interfere in our specific code-GRPO regime.

**Estimated GPU time: ~92 minutes.** ~50 lines of change.

### 3. `arxiv:2605.09608` — Geometry Conflict: Explaining and Controlling Forgetting in LLM Continual Post-Training
*May 10, 2026 — Wang et al. — **Deferred***

The paper explains our 200→400 task degradation: new training tasks whose gradient directions
are geometrically misaligned (negative cosine similarity) with the existing adapter's gradient
subspace cause representational conflicts that manifest as catastrophic forgetting. The framework
motivates a geometry-gated training extension: warm-start from the best 200×4 adapter, then
screen each of the 101-300 range tasks by their gradient cosine similarity to the best adapter's
training gradient, admitting only tasks with cosine_sim > 0.5.

**Why deferred:** Requires computing the Jacobian/gradient direction for 100 candidate tasks
relative to the existing adapter, which involves a forward+backward pass per task without
any reward feedback — pure gradient geometry analysis. This is ~80 lines of infrastructure
(gradient snapshot + cosine similarity computation) that should wait until C-CHAIN (EXP-027)
and the KL-penalty experiments clarify whether the bottleneck is distributional (KL) or
geometric (Jacobian). If C-CHAIN shows no improvement at 300 tasks, the geometry-gated
experiment becomes the next diagnostic.

200 tasks, 4 rollouts, lr=5e-6, LoRA r=16, binary eval@100

**Rationale:** arxiv:2605.06650 (POPO, Xu and Fang, May 7 2026) trains exclusively on
positive-rollout groups by showing that bounded importance sampling implicitly encodes
negative gradients through probability mass redistribution — no explicit negative rollout
is needed. For our 1.5B MBPP setup: at each training step, only groups with ≥1 passing
rollout produce gradient updates; the ~73% of all-zero groups (53% all-fail + 20% all-pass)
are skipped dynamically. The implicit negative signal emerges: when the IS-corrected policy
update increases P(positive rollout), probability conservation ensures P(negative rollout)
decreases implicitly. As training progresses, some previously all-fail tasks may produce a
positive rollout under the improved policy and become POPO-active — unlike EXP-039's static
pre-exploration filter. Key distinction: POPO requires zero infrastructure overhead (no
pre-exploration phase, no gradient cosine similarity, no AST parsing) — it is the simplest
possible selective-training approach. **~85 minutes.**

**Distinction from existing queue:**
- vs. DAPO (EXP-013): DAPO dynamically resamples until groups are mixed-outcome; POPO never
  uses negative rollouts explicitly, training only on positive groups as-is. ✓
- vs. EXP-039 (unlearnability filter): Static pre-exploration filter before training;
  POPO uses the current policy's natural positive-rollout distribution dynamically. ✓
- vs. RLOO (EXP-021): RLOO uses leave-one-out on all rollouts; POPO trains on positives-only
  with IS correction. ✓
- No prior experiment eliminates negative-rollout gradient contributions dynamically. ✓
