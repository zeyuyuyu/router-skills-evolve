## Top 3 Relevant Papers

### 1. "Selective-Advantage Entropy-Adaptive Horizon GRPO: Asymmetric Token-Level Discounting for Efficient Reinforcement Learning of Language Models" ★★★
**ArXiv ID**: 2606.05434 | June 3, 2026

**Core finding**: Introduces two extensions to GRPO. **(1) AH-GRPO**: weights each token's
policy gradient by a cumulative entropy-based discount factor γ_t = exp(−τ · Σ_{k≤t} H_k),
where H_k is the per-token entropy of the next-token distribution. This reduces the effective
training horizon for high-uncertainty token positions — the model is penalized less for tokens
where it is confused, and penalized more for tokens where it should be confident. **(2) SA-AH-GRPO
(Selective)**: applies this discount **only to negative-advantage rollouts** (failed trajectories),
leaving positive-advantage successful trajectories fully weighted. Tested on Qwen2.5-1.5B-Instruct
and Qwen2.5-3B-Instruct with LoRA on GSM8K mathematical reasoning. Reports +2.3 pp accuracy at
1.5B scale and faster convergence vs. vanilla GRPO.

**Why relevant to our system**: (a) Our exact base model (Qwen2.5-1.5B-Instruct) is one of the
paper's two evaluation models, making the results directly comparable. (b) The asymmetric selective
variant addresses the quality of GRPO gradient from non-collapsed groups (the 47.6% where variance
is nonzero), which is distinct from all queued ACR-mitigation experiments that target collapsed
groups. (c) For HumanEval code, the entropy profile is naturally structured: low entropy on
variable names and syntax (decisive choices), high entropy on algorithmic body (exploration) —
exactly the pattern where entropy-adaptive horizon should concentrate gradient on the correct parts.

**Motivation for EXP-098**: Run SA-AH-GRPO with τ=0.5 (paper default) for 1 GRPO cycle on
HumanEval with the standard DAPO config. Compare pass@1 and effective gradient horizon against
the DAPO binary baseline. If pass@1 improves, it shows that gradient QUALITY from non-collapsed
groups matters independently of ACR — a new finding for the AAAI paper.

### 2. "Advantage Collapse in Group Relative Policy Optimization: Diagnosis and Mitigation" ★★★
**ArXiv ID**: 2605.21125 | May 21, 2026 | (already queued as EXP-094)

Included here as context for the ACR-mitigation landscape. AVSPO (EXP-094) attacks ACR at the
group-sampling level via virtual sample injection. SA-AH-GRPO (EXP-098, today) attacks it at the
token-gradient level via entropy discount. These approaches are orthogonal: AVSPO creates
synthetic group diversity when real diversity is absent; SA-AH-GRPO improves gradient signal
quality from groups that already have real diversity. Running them sequentially (AVSPO first,
then SA-AH-GRPO, then a combination) is the correct evaluation order.

---

### 3. "Curriculum Reinforcement Learning from Easy to Hard Tasks Improves LLM Reasoning" ★★
**ArXiv ID**: 2506.06632 | March 2026 (last revised)

**Core finding**: E2H Reasoner schedules GRPO training tasks from easy (high zero-shot pass rate)
to hard (low zero-shot pass rate). Provides finite-sample complexity bounds showing curriculum
sampling requires fewer total samples than uniform sampling to reach the same accuracy level.
Reports improved pass@1 on mathematical reasoning at 7B scale.

**Why relevant as context**: The `grpo_curriculum_continual` kind in our runner.py is designed
for this curriculum pattern. However, for HumanEval at 1.5B scale with a 75.6%/24.4% label split
(mostly easy), a pure easy-to-hard curriculum would spend most of training on tasks the small model
already solves, which is wasteful. The more targeted ACR interventions (AVSPO, SA-AH-GRPO,
length penalty) are better use of the queue cap slots. E2H Reasoner is filed as context for
future curriculum experiments once ACR is resolved.

| Field | Value |
|---|---|
| Kind | `grpo_continual` |
| Priority | 8 |
| Base model | Qwen/Qwen2.5-Coder-1.5B-Instruct |
| Bench | HumanEval |
| N cycles | 1 |
| GRPO algo | `sa_ah_grpo` |
| Entropy discount τ | 0.5 (paper default for Qwen2.5-1.5B) |
| Selective (negative-only) | True |
| Dynamic sampling | True |
| N generations | 8 |
| GRPO temperature | 1.0 |
| Novel fields | `grpo_algo="sa_ah_grpo"`, `entropy_discount_tau=0.5`, `selective_negative_only=True` |
| Wall-clock est. | ~1.5 GPU-hours |
| GPU | auto (1× A800 80GB) |

**Rationale**: SA-AH-GRPO (arxiv:2606.05434) applies cumulative entropy-based discount to
token-level policy gradients, asymmetrically: only negative-advantage (failed) rollouts are
discounted, while positive-advantage (correct) rollouts remain fully weighted. This concentrates
gradient from failed trajectories on decisive, low-entropy token positions (variable names,
syntax) rather than uncertain, high-entropy positions (algorithmic body). Orthogonal to all
queued ACR-mitigation experiments: STARE (EXP-090) reweights advantages, AVSPO (EXP-094)
injects virtual samples, EXP-096 modifies the reward function. SA-AH-GRPO modifies the
**gradient weight** asymmetrically. Tested on our exact model (Qwen2.5-1.5B-Instruct) in the
paper; τ=0.5 is the paper's validated default.

**Non-duplication**: No queued experiment has `grpo_algo="sa_ah_grpo"` or `entropy_discount_tau`.
EXP-090 (STARE): operates on advantage scalars, not gradient weights. EXP-094 (AVSPO): group-level
sampling, not token-level discounting. EXP-096: reward function, not gradient computation. ✓

experiment must run as soon as the A800 comes back online (priority 9 = highest). 1-cycle ×

without running the full 4-cycle stack (which would take ~18 GPU-hours and might miss the
AAAI revision window). The 3 seeds [42, 123, 999] are conventionally used in the GRPO
literature; EXP-093's antiforgetting staircase used a different multi-seed structure (varying
training objective across seeds, not fixed-config variance). ✓
